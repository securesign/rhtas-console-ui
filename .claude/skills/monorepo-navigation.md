# Monorepo Navigation

## Workspaces

| Workspace | Package Name | Purpose | Build Tool |
|-----------|-------------|---------|------------|
| `common/` | `@console-ui/common` | Shared env config, branding | Rollup |
| `client/` | `@console-ui/client` | React SPA (PatternFly, Vite) | Vite |
| `server/` | `@console-ui/server` | Express production server | Rollup |
| `e2e/` | `@console-ui/e2e` | Playwright tests | — |

## Build Order

`common` must build before `client` or `server`. Root scripts enforce this:

```bash
# Build: common first, then client
npm run build    # "npm run build -w common && npm run build -w client"

# Test: common first, then all
npm test         # "npm run build -w common && npm run test -w common -w client -w server"

# Dev: common build, then concurrent dev servers
npm run start:dev
```

If you see "module not found" for `@console-ui/common`, run `npm run build -w common` first.

## Workspace-Specific Commands

Target a workspace with `-w`:

```bash
npm test -w client -- path/to/test.tsx   # single test file
npm run build -w common                  # rebuild common only
npm run generate -w client               # regenerate OpenAPI client
npm run start:dev -w client              # client dev server only
```

## Path Aliases

`@app/*` resolves to `client/src/app/*`. Configured in two places (both must agree):
- `client/tsconfig.app.json` — TypeScript resolution
- `client/vite.config.ts` — Vite bundler resolution

This alias is client-only. Other workspaces use standard relative imports.

## Cross-Workspace Imports

`client` and `server` import from `common` by package name:

```typescript
import { buildConsoleEnv, decodeEnv } from "@console-ui/common";
import { brandingStrings } from "@console-ui/common";
```

Neither lists `@console-ui/common` in their package.json dependencies — npm workspaces hoisting handles resolution. Don't add it as a dependency.

## Branding Pipeline

Branding flows through a non-obvious path:

1. Source: `branding/strings.json` (repo root)
2. Common rollup plugin resolves virtual module `@branding/strings.json`
3. Builds to `common/dist/branding/`
4. Client vite config locates branding via `require.resolve("@console-ui/common/package.json")`

To modify branding text, edit `branding/strings.json` and rebuild common.

## Generated Code (Do Not Edit)

`client/src/app/client/` contains auto-generated OpenAPI types and SDK. These files are regenerated by `npm run generate -w client`. Edit the spec at `client/openapi/console.yaml` instead.

## Linting & Formatting

Single root ESLint flat config (`eslint.config.mjs`) covers all workspaces. Prettier config at `prettier.rc.mjs`. Run from root:

```bash
npm run lint      # all workspaces
npm run format    # all workspaces
```

## Server in Production

Server serves pre-built client from `client/dist/`. Path hardcoded as `../../client/dist` relative to server source. If client hasn't been built, server will fail to serve the UI.
